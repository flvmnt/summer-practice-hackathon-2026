## Inline-string sneak (Phase 1-5 partial residue)

Pages that **do** use `setRequestLocale` but **do not** use `useTranslations`/`getTranslations`, instead defining a local `const COPY = { en, ro }` map:

- `src/app/[locale]/settings/page.tsx:17+` - full settings page copy
- `src/app/[locale]/calendar/page.tsx:17+` - duplicates `messages/*/common.json#calendar.*`
- `src/app/[locale]/groups/page.tsx:17+` - duplicates `groupsList.*`
- `src/app/[locale]/events/page.tsx` - duplicates `eventsList.*`

## Summary

The plumbing is solid: routing config, `proxy.ts` (Next 16 rename of middleware), session+DB locale columns, parity-enforced `lint:i18n`, and a dual-locale catalog with 393 keys per locale that pass the parity gate. The Wave 1+2 consolidation commit landed cleanly and removed all partials.

The shipping product, however, is not feature-complete on i18n: ~7 routes carry inline `COPY = { en, ro }` constants that duplicate (and can drift from) the catalog; the landing page and ~12 components carry hardcoded English; the `/today` language switch that gates the multi-language rubric bonus does not exist; and there is no authenticated locale-change action. The catalog passes parity, but the UI does not yet honor it everywhere.

Verdict: foundation **DONE**; sweep **PARTIAL**; multi-language bonus proof **MISSING**.
